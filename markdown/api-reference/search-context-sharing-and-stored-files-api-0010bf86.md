# Search, context, sharing, and stored files API

Created: 2026-05-29T17:20:37Z

## Note

This page covers endpoints that retrieve existing knowledge.

## Search notes

```text
GET /api/search
```

Authentication: session cookie or bearer token with `notes\:read` or `admin`.

Query parameters:

| Parameter | Required | Description |
|---|---:|---|
| `q` | Yes | Search query. Minimum length is 1. |
| `limit` | No | Maximum matches. Defaults to `50`; maximum `100`. |

Example:

```text
GET /api/search?q=product%20launch&limit=10
```

Response shape:

```json
[
  {
    "note": {
      "id": "4a95f146-9d27-4c79-b7d8-34739aef8998",
      "title": "Product launch meeting notes",
      "status": "ready"
    },
    "line_number": 12,
    "column": 8,
    "excerpt": "We discussed the product launch plan and next actions."
  }
]
```

Mianotes searches saved Markdown files with `ripgrep` and joins file matches back to note metadata.

## Get context

```text
GET /api/context
```

Authentication: session cookie or bearer token with `notes\:read` or `admin`.

This endpoint retrieves Markdown context from notes in a specific folder. It is designed for agents following prompts such as:

```text
Before answering, get context from the Settings notes in the Docs workspace.
```

Query parameters:

| Parameter | Required | Description |
|---|---:|---|
| `folder` | Yes | Folder name or slug. |
| `title` | Yes | Note title or context phrase. |
| `limit` | No | Maximum notes. Defaults to `5`; maximum `20`. |

The service first looks for matching note titles in the requested folder. If no title match is found, it searches Markdown files in that folder and returns matching notes.

## Share note

Create or replace a read-only share URL:

```text
POST /api/notes/{note_id}/share
```

Authentication: session cookie or bearer token with `share\:write` or `admin`.

Response:

```json
{
  "share_url": "http://127.0.0.1:8200/api/notes/shared/workspaces/default/NAC0tW5f..."
}
```

The returned URL includes the workspace id. This lets Mianotes resolve the shared note directly from the correct workspace database instead of searching across every configured workspace.

Disable sharing:

```text
DELETE /api/notes/{note_id}/share
```

## Get shared note

```text
GET /api/notes/shared/workspaces/{workspace_id}/{token}
```

Authentication: none.

Returns a shared note without requiring a session or bearer token.

`workspace_id` is the configured workspace id, not the workspace name shown in the UI. Invalid workspace ids or invalid share tokens return `404`.

Legacy token-only share URLs are not workspace-addressed and should not be generated by new clients.

## Get shared source file

```text
GET /api/notes/shared/workspaces/{workspace_id}/{token}/files/{source_file_id}
```

Authentication: none.

Returns a source file attached to a shared note.

## Get shared author avatar

```text
GET /api/notes/shared/workspaces/{workspace_id}/{token}/avatar
```

Authentication: none.

Returns the shared note author's avatar when the shared note can be resolved and the author has an avatar.

## Get published static HTML

```text
GET /html
GET /html/{file_path}
```

Authentication: none.

Returns generated static HTML files from published sites. The `/html` root
redirects to the latest published version.

## Get published Markdown

```text
GET /markdown/{file_path}
```

Unauthenticated callers can only read Markdown or source files for notes marked
as published.

## Get note-owned stored files

Stored file access is database-addressed. Mianotes resolves the workspace,
note, and source-file record before serving bytes.

```text
GET /api/workspaces/{workspace_id}/notes/{note_id}/markdown
GET /api/workspaces/{workspace_id}/notes/{note_id}/source-files/{source_file_id}
GET /api/workspaces/{workspace_id}/notes/{note_id}/images/{file_path}
```

Authentication: session cookie or bearer token with `notes\:read` or `admin`.

These routes only return files that belong to the requested note in the
requested workspace. Database files and arbitrary workspace files are never
served.
